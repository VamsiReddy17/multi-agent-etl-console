# Security Policy

We take the security of our Multi-Agent ETL Console architecture seriously. If you believe you have found a security vulnerability, please report it to us responsibly using the steps detailed below.

---

## Supported Versions

Only the current branch is actively supported with security updates:

| Version | Supported |
|---------|-----------|
| 1.0.x   | Yes       |
| < 1.0   | No        |

---

## Reporting a Vulnerability

**Please do not report security vulnerabilities via public GitHub issues.**

Instead, please send an email to the lead engineer or maintainers:
* **Contact:** [vamsirajareddy72@outlook.com](mailto:vamsirajareddy72@outlook.com)

Please include:
1. The type of issue (e.g., buffer overflow, sql injection, privilege escalation).
2. Step-by-step instructions or proof of concept code to reproduce the issue.
3. Potential impact of the vulnerability.

We will acknowledge receipt of your vulnerability report within **48 hours** and provide regular status updates during resolution.
